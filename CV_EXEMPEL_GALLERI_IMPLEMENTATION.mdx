### ✅ UX-FÖRBÄTTRINGAR (2025-01-24)

- Konsulterade SEO-agent och mobile-UX-designer för optimal struktur

**Implementerade förbättringar:**

1. **CV Preview Layout** (`CVExempelPage.tsx:89`)
   - **Före:** `max-w-6xl` (1280px) → För mycket white-space
   - **Efter:** `max-w-[900px]` → Optimal läsbredd, balanserad layout
   - **Resultat:** Eliminerad onödig white-space, bättre visuell täthet

2. **Tips Section Count** (`TipsSection.tsx:38,49`)
   - **Före:** Statisk text "Expertråd"
   - **Efter:** Dynamisk count `{tips.length} expertråd`
   - **Resultat:** Tydligare value proposition för användaren

3. **Varför det Fungerar - Text Density** (`VarforDetFungerar.tsx:89`)
   - **Före:** Default 16px text-base
   - **Efter:** `text-[15px]` för bättre densitet
   - **Resultat:** Mer scannable cards, bättre mobile UX

4. **Content Visibility Strategy** (baserat på SEO-agent rekommendation)
   - **Hero + SEO Intro:** 100% synlig (150-200 ord)
   - **Varför det fungerar:** 100% synlig, 6 cards (480-720 ord)
   - **Tips:** Accordion, 6 tips (600-900 ord)
   - **FAQ:** Accordion, 10+ frågor (1000-1500 ord)
   - **Total:** 3500-4000 ord per sida, 50% synlig / 50% expanderbar

5. **Schema.org FAQ Fix** (både cv-exempel och personligt-brev-exempel)
   - **Problem:** Google Search Console rapporterade missing 'name' och 'text' fields
   - **Root Cause:** Tomma FAQ-entries genererade ogiltig Schema.org markup
   - **Fix:** Lade till `.filter()` för att ta bort entries med tomma fält
   ```typescript
   mainEntity: faq
     .filter(item => item.fraga && item.svar && item.fraga.trim() !== '' && item.svar.trim() !== '')
     .map(item => ({
       '@type': 'Question',
       name: item.fraga,
       acceptedAnswer: {
         '@type': 'Answer',
         text: item.svar
       }
     }))
   ```

**Design Principer (från mobile-UX-designer):**
- Mobile-first: Touch-friendly accordion buttons (min-height: 56-64px)
- Scannable: Headers alltid synliga, endast content expanderbar
- Progressive Disclosure: Viktigt content synligt, detaljer bakom accordion
- Visual Hierarchy: Gradient badges, numbered tips, icon-system
- Consistent Spacing: 16px mobile → 24px desktop margins

## Skrivguide för CV-exempel

### Varför det fungerar (6 punkter)

**Syfte**: Pedagogisk, förklara VARFÖR exemplet är bra

**Format**:
```typescript
{
  rubrik: "Kort, påståendefull rubrik",
  text: "1-2 meningar förklaring. Ska vara konkret och educational."
}
```

**Teman att täcka**:
1. ATS-optimering (nyckelord, struktur)
2. Kvantifierbara resultat (siffror, mätbart)
3. Balans teknisk/mjuk kompetens
4. Certifieringar/trovärdighet
5. Profiltext som säljande sammanfattning
6. Progression/ansvar

### Tips (6 punkter)

**Syfte**: Actionable, praktiska råd för läsaren

**Format**:
```typescript
{
  rubrik: "Imperativ formulering - vad läsaren ska göra",
  text: "2-3 stycken (separerade med \n\n). Första stycket: varför. Andra stycket: hur. Tredje stycket: exempel."
}
```

**Teman**:
1. Branschspecifika nyckelord
2. Kvantifiera erfarenhet
3. Visa konkreta resultat (inte bara ansvar)
4. Anpassa profiltext per jobb
5. Lyft fram certifieringar
6. Balansera tekniska/mjuka färdigheter med bevis

**Tips-stil**:
- Inga tankstreck som skiljetecken
- Actionorienterat: "Inkludera...", "Visa...", "Anpassa..."
- Konkreta exempel: "Istället för X, skriv Y"
- Multi-paragraph för tips >80 ord (använd \n\n)

### FAQ (10 frågor) - KRITISK STRATEGI

**⚠️ USER FEEDBACK:**
> "FAQ:n, om vi har så här generiska frågor som inte är kopplat till undersköterska...hur gör vi då när vi ska ha 30st sidor med olika yrken? Borde dem inte vara nischade till det specifika yrket?"

**LÖSNING: Hybrid-strategi (3-4 generiska + 6-8 yrkes-specifika)**

#### Generiska Frågor (3-4 st - återanvändbara för alla yrken)

Dessa frågor är universella och kan användas på alla CV-exempel sidor:

1. **"Hur långt ska mitt CV vara?"**
   - Svar: "I Sverige rekommenderas 1-2 sidor. För juniora roller (0-5 års erfarenhet) räcker vanligtvis 1 sida, medan seniora yrkespersoner kan behöva 2 sidor för att visa progressionen i sin karriär."

2. **"Ska jag ha en profilbild på mitt CV?"**
   - Svar: "I Sverige är det vanligt men inte obligatoriskt. Välj en professionell bild med neutral bakgrund och affärsmässig klädsel. Undvik semesterbilder eller selfies."

3. **"Hur hanterar jag luckor i mitt CV?"**
   - Svar: "Var ärlig och kort om luckor. Vid kortare uppehåll (< 6 månader) behöver du inte förklara. För längre perioder, nämn konstruktiva aktiviteter som vidareutbildning, volontärarbete eller föräldraledighet."

4. **"Ska jag skicka samma CV till alla jobb?"** (VALFRI - kan bytas mot yrkes-specifik)
   - Svar: "Nej, anpassa alltid ditt CV för varje tjänst. Justera nyckelord, lyft relevant erfarenhet och matcha din kompetens mot kravprofilen. Detta ökar chansen att passera ATS-system och imponera på rekryterare."

#### Yrkes-Specifika Frågor (6-8 st - MÅSTE vara unika per yrke)

**Template för att skapa yrkes-specifika frågor:**

**Kategori 1: Bransch-specifika kompetenser/certifieringar**
- Exempel Undersköterska: "Ska jag lista medicinsk delegering på mitt CV som undersköterska?"
- Exempel Lagerarbetare: "Ska jag nämna mitt truckkort på CV:t?"
- Exempel IT-konsult: "Hur många programmeringsspråk ska jag lista?"
- Exempel Lärare: "Ska jag inkludera VFU-placeringar i mitt CV?"

**Kategori 2: Praktisk yrkesutövning**
- Exempel Undersköterska: "Hur visar jag att jag kan hantera fysiskt krävande arbete?"
- Exempel Receptionist: "Hur beskriver jag erfarenhet av olika telefonsystem?"
- Exempel Kock: "Hur visar jag erfarenhet från olika typer av kök?"
- Exempel Administrator: "Ska jag lista alla administrativa system jag kan?"

**Kategori 3: Arbetsmiljö/setting**
- Exempel Undersköterska: "Hur anpassar jag CV:t för olika vårdmiljöer (äldreboende vs hemtjänst)?"
- Exempel Butiksbiträde: "Hur visar jag erfarenhet från både stora kedjor och små butiker?"
- Exempel IT-konsult: "Hur beskriver jag remote vs on-site konsultuppdrag?"

**Kategori 4: Mjuka färdigheter (yrkes-specifikt beskrivet)**
- Exempel Undersköterska: "Hur visar jag att jag är bra på teamarbete i vården?"
- Exempel Servitris: "Hur beskriver jag min stresstolerans under rusningar?"
- Exempel Chef: "Hur visar jag ledarskapsförmåga utan att framstå som skrytsam?"

**Kategori 5: Karriärprogression i branschen**
- Exempel Undersköterska: "Ska jag lista alla vårdgivare jag jobbat för?"
- Exempel Handläggare: "Hur visar jag progression inom offentlig sektor?"
- Exempel Projektledare: "Hur beskriver jag projektportfölj vs individuella projekt?"

**Kategori 6: Bransch-specifika utmaningar**
- Exempel Förskollärare: "Hur beskriver jag erfarenhet från små vs stora barngrupper?"
- Exempel Ekonomiassistent: "Ska jag nämna vilka ekonomisystem jag kan?"
- Exempel Lokalvårdare: "Hur visar jag erfarenhet från olika typer av miljöer?"

#### Implementation Checklist för Varje Yrke

När du skapar FAQ för ett nytt yrke:

1. ✅ Kopiera 3-4 generiska frågor (samma för alla yrken)
2. ✅ Skapa 6-8 yrkes-specifika frågor enligt kategorierna ovan
3. ✅ Researcha branschterminologi (t.ex. "Cosmic" för undersköterskor)
4. ✅ Inkludera long-tail keywords (t.ex. "cv undersköterska medicinsk delegering")
5. ✅ Testa att frågorna faktiskt är unika (gör Google-sökning)
6. ✅ Verifiera Schema.org markup (inga tomma fält)

## Troubleshooting

### Build errors

**Problem**: `params is not a Promise`
**Solution**: Next.js 15 kräver `params: Promise<{ yrke: string }>` och `await params`

**Problem**: SSG inte aktiverat
**Solution**: Verifiera att `generateStaticParams()` finns och returnerar alla yrken

### Preview issues

**Problem**: SVG templates visas inte
**Solution**: Kontrollera att `imagePath` i `SIMPLE_TEMPLATES` matchar filer i `/public/mallar/*.svg`

**Problem**: Mobile bottom sheet stängs inte
**Solution**: Verifiera click-outside detection är disabled på mobile (`!isMobile` check)

## Quick-Start Guide: Skapa Nytt Yrke (30 min per yrke)

**Förutsättningar:**
- 1 yrke redan implementerat (Undersköterska) som referens
- Tillgång till branschkunskap eller research om yrket
- Sökvolym-data för prioritering

**Steg-för-steg process:**

### Steg 1: Data Research (10 min)
1. Researcha branschterminologi (t.ex. "Cosmic" för undersköterskor)
2. Identifiera 5-7 nyckelkompetenser för yrket
3. Lista 3-5 vanliga certifieringar/utbildningar
4. Hitta 2-3 konkreta resultat-exempel (med siffror)

### Steg 2: Skapa exempelCV (10 min)
```typescript
// I src/app/cv-exempel/[yrke]/page.tsx
const exampleData = {
  'nytt-yrke': {
    yrke: "Nytt Yrke",
    sokvolym: 750,
    kategori: 'vard', // eller 'teknik', 'service', etc.

    exempelCV: {
      namn: "Förnamn Efternamn",
      titel: "Professionell Titel",
      kontakt: { /* ... */ },
      profil: "3-4 meningar med nyckelord + mätbara resultat",
      erfarenhet: [
        {
          titel: "Senior Position",
          arbetsgivare: "Företag AB",
          period: "2020 – Pågående",
          beskrivning: [
            "Kvantifierbart resultat med siffror (t.ex. 'minskade X med 25%')",
            "Specifik branschterminologi och verktyg",
            "Ledarskap/ansvar (om relevant)"
          ]
        }
      ],
      // ... fortsätt enligt data structure
    }
  }
}
```

### Steg 3: Skapa Content (10 min)

**3.1 SEO Intro (150-200 ord)**
- Stycke 1: Vad utmärker ett bra CV för detta yrke?
- Stycke 2: Branschspecifika krav (ATS, nyckelord, certifieringar)
- Stycke 3: Vad exemplet visar (konkreta styrkor)

**3.2 Varför det fungerar (6 punkter, 80-120 ord vardera)**
- Kvantifierbara resultat
- ATS-optimering
- Branschspecifika nyckelord
- Certifieringar/trovärdighet
- Balans teknisk/mjuk kompetens
- Progression/ansvar

**3.3 Tips (6 punkter, 100-150 ord vardera)**
- Använd imperativ form ("Inkludera...", "Visa...", "Anpassa...")
- Multi-paragraph med \n\n för 100+ ord
- Konkreta exempel: "Istället för X, skriv Y"

**3.4 FAQ (3-4 generiska + 6-8 yrkes-specifika)**
- Kopiera generiska från Undersköterska-exemplet
- Skapa yrkes-specifika enligt kategorierna i FAQ-strategin ovan
- Testa uniqueness med Google-sökning

**3.5 Relaterade Yrken (3 st)**
- Välj yrken i samma kategori eller med överlappande kompetenser

### Steg 4: Verifiera & Testa (5 min)
```bash
# Bygg och testa lokalt
npm run build

# Verifiera att sidan genereras
# Sök efter "● /cv-exempel/[nytt-yrke]" i build output

# Testa i browser
npm run dev
# Besök http://localhost:3000/cv-exempel/nytt-yrke
```

**Checklist innan commit:**
- ✅ CV-data fylld med branschspecifik information
- ✅ Alla beskrivningar har kvantifierbara resultat
- ✅ FAQ innehåller 60%+ yrkes-specifika frågor
- ✅ Schema.org validering (inga tomma FAQ-fält)
- ✅ Relaterade yrken länkade korrekt
